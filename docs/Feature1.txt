Feature request:
When a user taps Finish Workout, the app should check whether the workout is fully completed before ending it. If any set has not been marked complete with the checkmark, or if any weight or reps field is still empty, show a confirmation dialog warning the user that the workout has incomplete sets or missing inputs and ask whether they still want to finish. If everything is complete, the existing finish flow should continue normally without the extra warning.

User outcome:
Users get a clear warning before accidentally finishing a workout that still has unchecked sets or missing weight/reps values, while still being allowed to finish intentionally if they choose to.

Acceptance criteria:
- Tapping Finish Workout checks all exercises and sets in the active workout.
- If any set is unchecked, or if weight or reps is blank for any set, show a confirmation dialog before finishing.
- The dialog clearly tells the user the workout is incomplete and offers options to go back or finish anyway.
- If all sets are checked and all weight/reps fields are filled, finishing the workout behaves as it does today with no extra warning.
- This behavior works for both regular workouts and quick workouts.

Non-goals:
- Do not redesign the active workout screen.
- Do not change how completed workouts are stored beyond this validation/confirmation behavior.
- Do not require users to fill values before finishing; they should still be allowed to override and finish anyway.
- Do not add cloud sync, analytics, or new dependencies.
